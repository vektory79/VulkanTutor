# vulkan-testing

**Правила тестирования библиотеки-обёртки Vulkan.**

## Когда применять

- При добавлении новой обёртки (структуры, флага, класса)
- При изменении существующей обёртки
- При написании нового урока, использующего обёртку

## Правила

### 1. Только интеграционные тесты

Тестируем **корректность работы обёрток через реальные вызовы Vulkan API**. Не пишем тесты на отдельные геттеры, сеттеры, конструкторы — это тестирование LWJGL.

### 2. Наследовать от VulkanTestBase

Все интеграционные тесты наследуют `VulkanTestBase` из `src/test/kotlin/me/vektory79/vulkan/kotlin/`. GLFW инициализируется один раз на класс, а не на каждый тест.

### 3. AssertJ для проверок

Используем `assertThat` вместо JUnit `assertEquals`/`assertTrue`.

### 4. Тегирование

- `@Tag("integration")` — тест требует GPU и Vulkan
- `@Tag("unit")` — тест без GPU (ошибки, маппинг кодов)

### 5. Очистка ресурсов

Каждый VK-ресурс (device, queue, surface) уничтожается в `finally`. Instance и window уничтожает `VulkanTestBase`.

### 6. Пайплайн, не изоляция

Тест проверяет цепочку вызовов: instance → physical device → logical device → queue, а не отдельный метод.
