# confirm-before-implementing

Do not start implementing until the user confirms the approach.

- When a task involves multiple steps or design decisions, outline the plan and wait for confirmation before writing code.
- When the user says "wait", "stop", or "actually", halt immediately. Do not continue the current approach. Ask what to do next.
- Do not start phase N+1 until the user has confirmed phase N is complete.
- Do not switch branches, create files, or run mutations while the user is still thinking through the design.
